# e662. 11830 - Contract Revision

## 內容

多年以來，the Association of Contracts for Modernization(現代化合同協會ACM)的所有合同都是使用舊打字機打字的。
最近，一位ACM的會計Miranda先生，他意識到該機器有一位數字故障，並且只有一位數字故障。
更具體地說，故障的數字在打字時不會印在紙張上，就像沒有按下按鍵一樣。
他意識到這可能會改變合同價值。由於擔心財務問題，Miranda先生想從合同約定的原始價值中，知道合同在打錯後實際代表的價值。
例如：假設機器中的故障數字為5，由於不會打印數字5，因此在相應的合同中，會將1500的約定值打成為100。
請注意，Miranda先生想知道合同中表示的"數值"，所以數字5000對應於數值0，而不是000。

## 輸入說明

輸入包含多組測資，每組測資一行。
每行包含兩個整數D (1 ≤ D ≤ 9)和N (1 ≤ N < 10^100)，分別代表機器中發生故障的數字和最初合同約定的數字(可能非常大)。
如果 D = N = 0代表輸入結束。

## 輸出說明

對於每組測資，輸出合同在打錯後實際代表的價值。

## 範例輸入 #1

```
5 5000000
3 123456
9 23454324543423
9 99999999991999999
7 777
0 0
```

## 範例輸出 #1

```
0
12456
23454324543423
1
0
```
