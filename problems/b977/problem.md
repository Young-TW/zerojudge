# b977. 6.最終任務->b.交換提示

## 內容

找到提示後，各組開始解謎，可是每組都不會拿完提示，這時主持人公布你可以和別組交換提示，你帶著又喜又懼的心情和對方交換，因為你怕換到你拿過的，於是你決定寫一個程式來判斷和哪一組有比較多機會換到不一樣的

## 輸入說明

第一行由正整數n,m,q代表有n組，m總提示及q筆詢問(0<n,m<=1000,0<q<=1000)，接下來n行每行有一個數字a(a<=m)及a個數字，a代表各組有的提示種類，a個數字代表總類編號(1~m)，接下來有q行的數字(s,t)詢問第s和第t組共有幾種換法可以讓雙方得到沒拿過的提示，請參考範例輸入

## 輸出說明

根據每筆尋問，輸出換法數，請參考範例輸出

## 範例輸入 #1

```
4 5 3
1 1
2 1 2
3 1 2 5
2 2 4
1 3
2 4
1 4
```

## 範例輸出 #1

```
0
1
2
```
