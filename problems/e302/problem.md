# e302. PI. Inherit Previous Exam(繼承考古題)

## 內容

⼩明是個⺟湯⼤學的⼤學⽣，他⼀直覺得很奇怪，為什麼同學的作業都可以這麼快寫完，⽽⾃⼰怎麼都寫很久寫不完。後來⼩明才知道，原來⺟湯⼤學有⼀個考古題⾷物鏈，對該科⽬擅⻑的學⻑姊會作為考古題⽣產者，把作業和考試的答案交給⾃⼰的⼀個直屬學弟妹，⽽不擅⻑的⼈就會成會消費者，消耗掉該考古題。要知道教授要是發現你的程式碼是學⻑姊⽤過得，就會把你當掉，所以⼀份考古題只能⽤⼀次。

當然，如果學弟妹沒有⽤掉考古題，學⻑姊會很傷⼼，所以如果現在有多個學⻑姊留下來的考古題的話，那個⼈會選擇屆數離它最近的那⼀屆的考古題來⽤，畢竟屆數近⼈情壓⼒也⽐較⼤。如果還有剩下的考古題，它會把它繼續傳給⾃⼰的學弟妹。

現在給你⼀個直屬的⼀連串的學⽣資料，⼩明想知道，會不會有學弟妹沒有考古題可以⽤⽽被當掉，或是有學⻑姊的考古題沒⼈⽤⽽很傷⼼呢？

## 輸入說明

題⽬第⼀⾏有⼀個整數$T$，代表有幾組學⽣。接下來$T$⾏，每⾏有⼀個包含Y或N的字串$S$，其中$Y$代表這個學⽣會產⽣⼀份考古題，並把所有考古題傳給下⼀個學⽣，$N$代表這個學⽣會消耗⼀份考古題，並把剩下的傳給下⼀個學⽣。


$T \leq 10$
$|S| \leq 1.2 * 10^6$

## 輸出說明

對於每組學⽣，如過可以完美⽤完所有的考古題，且沒有⼈會被當掉，則輸出⼀⾏YES，否則輸出⼀⾏NO。

## 範例輸入 #1

```
4
YNYYNYNN
YYNNYYNN
YNYNYNYY
YNYNYNN
```

## 範例輸出 #1

```
YES
YES
NO
NO
```
