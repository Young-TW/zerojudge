# d901. NOIP2010 3.导弹拦截

## 內容

经过11 年的韬光养晦，某国研发出了一种新的导弹拦截系统，凡是与它的距离不超过
其工作半径的导弹都能够被它成功拦截。当工作半径为0 时，则能够拦截与它位置恰好相同
的导弹。但该导弹拦截系统也存在这样的缺陷：每套系统每天只能设定一次工作半径。而当
天的使用代价，就是所有系统工作半径的平方和。
某天，雷达捕捉到敌国的导弹来袭。由于该系统尚处于试验阶段，所以只有两套系统投
入工作。如果现在的要求是拦截所有的导弹，请计算这一天的最小使用代价。

## 輸入說明

第一行包含4 个整数x1、y1、x2、y2，每两个整数之间用一个空格隔开，表示这两套导
弹拦截系统的坐标分别为(x1, y1)、(x2, y2)。
第二行包含1 个整数N，表示有N 颗导弹。接下来N 行，每行两个整数x、y，中间用
一个空格隔开，表示一颗导弹的坐标(x, y)。不同导弹的坐标可能相同。

## 輸出說明

输出只有一行，包含一个整数，即当天的最小使用代价。

## 範例輸入 #1

```
输入样例1：
0 0 10 0
2
-3 3
10 0
输入样例2：
0 0 6 0
5
-4 -2
-2 3
4 0
6 -2
9 1
```

## 範例輸出 #1

```
输出样例1：
18
输出样例2:
30
```
