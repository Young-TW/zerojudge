# f281. 志樺的亮度

## 內容

志樺的班級打算全班來個大合照，但是志樺的膚色太深了，導致在照片中看不清楚他。我們可以透過調整照片的亮度來讓他變清楚，但是有個麻煩的問題 — 如果把照片的亮度增加了N，雖然志樺的明顯度會上升N，但其他同學的明顯度則會下降N。（若一個同學的明顯度越高，則他在照片中越能被清楚看見）
由於志樺曾經送你一隻馬島蟑螂當作寵物，你欠他一個人情，所以現在他想請你幫他寫個程式計算亮度最少要增加多少才可以讓所有同學的明顯度都不小於L。

## 輸入說明

每個測資點只有一組測資


第一行輸入一個正整數C，代表志樺班上共有C個同學。（C<=10^5）


第二行有C個以空白隔開的數字Mi，分別代表志樺斑上所有同學原本的明顯度，明顯度最低的即為志樺的明顯度。（0<=Mi<=10000）。


第三行有一個正整數L，L必大於志樺的亮度。（L<=10000）

## 輸出說明

如果照片不管增加多少亮度皆無法讓所有同學的明顯度都不小於L，則輸出「You are too black!」（不含引號）；否則請輸出一個正整數N代表照片最少要增加多少亮度。

## 範例輸入 #1

```
3
4 1 3 
2
```

## 範例輸出 #1

```
1
```

## 範例輸入 #2

```
4
1 3 2 2 
2
```

## 範例輸出 #2

```
You are too black!
```
