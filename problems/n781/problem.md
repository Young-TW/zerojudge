# n781. 10258－Contest Scoreboard

## 內容

覺得比賽的記分板有問題嗎？這是你機會來得出正確的排名。參賽者首先按解決問題的數量排名（越多越好），然後按懲罰時間的遞減排列。如果兩個或更多參賽者在解決問題數量和懲罰時間上均相同，則按隊伍號碼的遞增順序顯示。


如果參賽者對某個問題的任何提交被判定為正確，則該問題被視為解決。懲罰時間的計算是從首次正確提交所需的分鐘數加上每次正確提交之前的每次錯誤提交的20分鐘。不解決的問題不會產生時間懲罰。

## 輸入說明

輸入以一行單獨的正整數開始，表示接下來的測試案例數量。這行之後是空行，並且在兩個連續的輸入之間也有空行。


輸入包括判定隊列的快照，包含一些或全部參賽者（編號從1到100）解決問題（編號從1到9）的條目。每行輸入將由三個數字和一個字母組成，格式為：


contestant problem time L


其中 L 可以是 ‘C’, ‘I’, ‘R’, ‘U’ 或 ‘E’。這些分別代表 Correct（正確）, Incorrect（錯誤）, clarification Request（澄清請求）, Unjudged（未判定）和 Erroneous submission（錯誤提交）。後三種情況不影響計分。


輸入行按提交接收的順序排列。

## 輸出說明

對於每個測試案例，輸出必須遵循以下描述。兩個連續案例的輸出之間將以空行分隔。


輸出將包含按先前描述排序的記分板。每行輸出將包含參賽者編號、參賽者解決的問題數量以及參賽者累積的時間懲罰。由於並非所有編號從1到100的參賽者實際參與比賽，僅顯示那些提交過答案的參賽者。

## 範例輸入 #1

```
1

1 2 10 I
3 1 11 C
1 2 19 R
1 2 21 C
1 1 25 C
```
