# e390. 許功蓋問題

## 內容

有時寫程式會遇到一個問題，叫「許功蓋問題」


會有這個問題的原因就是：中文字占兩個字元或三個字元。在一些軟體上面，中文字占兩個字元 ( 像是 Dev C++ 和 CodeBlocks )，如果第二個字元編碼跟 '\' 一樣 ( 許、功、蓋就是這種字元 )，編譯器就會搞混，造成一些問題。不過如果像是 Zerojudge 中文字占三個字元的就不會發生這種問題。


現在要你先找出第二個字元編碼跟 '\' 一樣的中文字，然後在 Zerojudge 上面判斷是否為「許功蓋字元」。

## 輸入說明

一些非空白的字元(EOF結束)

## 輸出說明

如果是會造成許功蓋問題的字元


輸出 Yes


否則輸出 No

## 範例輸入 #1

```
許
C
蓋
a
1
我
豹
```

## 範例輸出 #1

```
Yes
No
Yes
No
No
No
Yes
```
