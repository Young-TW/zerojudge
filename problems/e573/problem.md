# e573. 10700 - Camel trading

## 內容

在西元後800年左右，巴格達的El Mamum提出了公式1 + 2 * 3 * 4 + 5，該公式起源於駱駝交易的財務帳戶。
該公式沒有括號，並且模棱兩可。因此，他決定請賢者為他提供一種方法，根據對駱駝的買賣來確定哪種解釋對他最有利。
El Mamum委託您寫一個程式，該程序需要確定無括號表達式可計算的最大和最小和。

## 輸入說明

輸入第一行有一個整數N，N代表有幾組測資。
接下來N行，每行包含一個表達式。
每個表達式最多由12個數字組成，每個數字介於1到20之間，並由運算符 + 和 * 隔開。

## 輸出說明

對於每組測資的表達式，計算此表達式可計算的最大和最小和。
並且按照格式輸出(請參考範例輸出)。

## 範例輸入 #1

```
3
1+2*3*4+5
4*18+14+7*10
3+11+4*1*13*12*8+3*3+8
```

## 範例輸出 #1

```
The maximum and minimum are 81 and 30.
The maximum and minimum are 1560 and 156.
The maximum and minimum are 339768 and 5023.
```
