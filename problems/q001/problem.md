# q001. "421"循環

## 內容

小明在網上看到了一種說法，就是如果你輸入一個數正整數a(a>2)，如果這個數是奇數，那就把它乘三倍在加一，如果這個數是偶數，那就把它除以2，經過n次操作，這個數會落入“421”循環，連科學家也無法解釋！小明想用編程來看看一個數需要多少此操作才能落入“421”循環，以尋找規律。


 



說明：這個數要多少次操作才能變成4


5->16->8->4         三次操作

## 輸入說明

多筆測資，每行一個整數a(a>2)

## 輸出說明

所需要的操作次數n

## 範例輸入 #1

```
53
```

## 範例輸出 #1

```
67
```

## 範例輸入 #2

```
67
```

## 範例輸出 #2

```
25
```

## 範例輸入 #3

```
4
```

## 範例輸出 #3

```
0
```
